## 데이터 저장 구조

### 랜딩 텍스트/구성 데이터

- Supabase `landing_contents` 테이블에 저장
- 저장 키는 `id = 'main'`

### 이미지 저장 방식

- 현재는 `Supabase Storage` 사용 구조로 변경
- 이미지 파일은 Storage 버킷에 업로드
- DB에는 이미지 파일 자체가 아니라 `public URL` 저장

기본 버킷 이름:

- `landing-assets`

## 추가로 Supabase에서 필요한 것

Storage 버킷은 생성 완료 상태.

추가 SQL 필요:

```sql
create table if not exists landing_contents (
  id text primary key,
  content jsonb not null,
  updated_at timestamptz not null default now()
);
```
